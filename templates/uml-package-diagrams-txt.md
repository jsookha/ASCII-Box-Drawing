### **Purpose**
Shows logical grouping of classes/modules into packages.  
Useful for large‑scale architecture or modular design.

---

## ✅ **Example 1 — Basic Package Structure**

```
┌──────────────────────┐
│     ui               │
├──────────────────────┤
│ LoginPage            │
│ Dashboard            │
└──────────────────────┘

┌──────────────────────┐
│   services           │
├──────────────────────┤
│ AuthService          │
│ UserService          │
└──────────────────────┘
```

---

## ✅ **Example 2 — Package Dependencies**

```
┌──────────────┐      ┌──────────────┐
│   domain     │◄──── │  services    │
└──────────────┘      └──────────────┘
        ▲
        │
┌──────────────┐
│   ui         │
└──────────────┘
```

## ✅ **Example 3 — Nested Packages**

```
┌──────────────────────────────────────────────┐
│                 app                          │
│ ┌──────────────────────────────────────────┐ │
│ │               domain                     │ │
│ │  ┌──────────────┐   ┌─────────────────┐  │ │
│ │  │  user        │   │  orders         │  │ │
│ │  └──────────────┘   └─────────────────┘  │ │
│ └──────────────────────────────────────────┘ │
└──────────────────────────────────────────────┘
```
